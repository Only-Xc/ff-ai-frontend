## Model Correction Notes

### How To Record Lessons

- 记录经验时聚焦纠正模型容易犯的判断错误，不复述已经写进代码的实现清单。
- 抽象成可迁移的判断规则：什么时候先排查什么、什么时候提升抽象、什么时候收窄职责边界。
- 避免记录具体文件名、函数名、属性列表和本次改动流水账，除非它们代表长期架构边界。
- 经验应帮助下一次做对决策，而不是帮助回忆这一次改了哪些代码。
- 用户指出“这不是经验”时，应把内容从实现细节改写为模型纠偏原则。

### AntD Table

- 遇到 Table 滚动条颜色、背景、空白、错位问题时，先排查传入 AntD 的参数、列宽、主题 token、容器尺寸和 fixed column 约束，再考虑样式覆盖。
- AntD 官方组件表现异常时，优先假设本项目的布局、主题或参数组合影响了组件表现。
- 横向滚动和 fixed column 同时出现时，重点检查列宽总和、`scroll.x`、固定列宽度和操作内容实际宽度。
- 看到数据较少时横向滚动条位置异常，优先判断 Table 内容区域高度是否被内容自然高度撑开。

### Abstraction Boundary

- 可复用 wrapper 的职责要窄：负责测量和注入派生属性，业务配置仍由子组件自己声明。
- hook 或 wrapper 的 API 要围绕调用方真实重复点设计，避免为了“通用”暴露会制造歧义的参数。
- 已经能从子组件 props 合并得到的信息，优先从子组件读取并合并，减少 wrapper 的重复入参。
- 对 AntD 这类复杂组件封装时，保留原生能力和原生样式，封装层只补齐布局约束。

### State Management

- 分页这类跨页面一致的状态，优先抽成单个对象返回，调用处通过一个变量访问 query、props 和行为方法。
- 跨页面一致的默认配置放进 hook 内部集中管理，页面只传业务差异。
- 筛选条件变化和 pageSize 变化会影响分页边界，分页 hook 需要统一处理回到第一页。

### Routing And Page Structure

- 详情页、看板页、编辑页这类独立任务页面应使用独立路由渲染，避免挂在列表页 Tabs 或局部 Outlet 区域里。
- 调整嵌套路由前先判断页面的视觉归属：列表切换页适合放在 tab 容器内，详情工作流页面适合提升到同级独立 route。
- 独立页面组件应放在页面目录外层；`components` 目录保留给列表行、筛选区、标签、格式化展示等可复用或局部组件。
- 提升隐藏详情路由时，要确认侧边栏高亮来源。可见菜单通常按路径前缀高亮，隐藏 route 只需要保持 URL 层级或显式 `navKey`。
- 保持 URL 不变时，优先调整 route 层级和渲染容器，避免为了页面归属改变已有导航链接。

### Implementation Hygiene

- 合并嵌套 props 时使用不可变合并，保持调用方传入对象稳定。
- 新增样式覆盖前先定位根因，尤其是第三方组件已有正常官方表现时。
- 修改布局类问题后，同时验证数据少、数据多、横向滚动、fixed column 和分页切换场景。
