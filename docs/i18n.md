## 任务：项目 i18n 国际化改造（保持语义，禁止硬翻译）

你是一个资深前端工程师，请帮我完成项目中的国际化文本抽离与翻译。项目技术栈为React + TypeScript + react-i18next。

### 一、抽离要求（同前）
1. 扫描所有组件/页面文件（`./src`），找出用户可见的硬编码中文字符串。
2. 忽略 console、注释、变量名、URL、正则、测试文件。
3. 为每个唯一文本生成语义化 key，格式如 `common.save`、`page.login.title`，同一文本复用 key。
4. 创建 `locales/zh-CN.json` 和 `locales/en-US.json`，保留原文为中文值。
5. 替换硬编码为 `t('key')` / `$t('key')` 等。

### 二、翻译质量要求（重点）
- **禁止字对字硬翻译**：不能将中文逐词替换为英文单词。例如：
  - ❌ “保存” → “preserve”（正确应为 “save”）
  - ❌ “记住密码” → “remember password word”（正确应为 “Remember password”）
- **必须保持语义一致**：理解中文原文在实际业务场景中的真实含义，再用地道英文表达。
- **符合目标语言习惯**：
  - 英文：注意大小写（标题首字母大写，按钮一般首字母大写）、单复数、冠词。
  - 其他语言（如日语、韩语）需遵循对应礼貌等级。
- **上下文优先**：遇到有歧义的词（如“新建”在文件管理 vs 聊天窗口），结合组件所在模块判断。
- **动态内容占位符**：若原文包含 `{0}`、`%s` 等变量，英文翻译中必须保留并放置于正确位置。

**示例对照：**
| 中文原文 | ❌ 硬翻译 | ✅ 语义翻译 |
|---------|-----------|-------------|
| 点击上传 | Click click upload | Click to upload |
| 当前无数据 | Currently no data | No data available |
| 确定删除？ | Determine delete? | Are you sure you want to delete? |
| 用户名或密码错误 | User name or password mistake | Incorrect username or password |

### 三、输出格式
对于每个修改的文件，输出：
- 原代码 → 替换后代码
- 新增的 i18n 条目（中英对照，且英文部分需经过语义审核）

最后生成变更清单（修改文件 + 新增 key 列表）。

请开始执行，遇到不确定语义的字符串请先提问。
