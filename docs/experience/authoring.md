# 经验维护规范

## 条目格式

```md
### EXP-TOPIC-000 标题

- 触发：任务中出现的具体信号。
- 经验：下一次应直接采用的判断或做法。
- 检查：执行前后要核对的关键点。
- 关联：需要补读的 topic；没有写“无”。
```

## 写作规则

- 每条经验只解决一个稳定判断问题。
- `触发` 写可匹配的任务信号，避免宽泛概念。
- `经验` 写可执行决策，避免理念描述。
- `检查` 控制在 3-5 个关键点。
- `关联` 只写真实需要补读的 topic。
- 能扩展旧经验时扩展旧经验。
- 临时问题、单次字段清单、一次性修复细节沉淀为可迁移规则。

## Topic 规则

- topic 文件面向运行时读取，保持短小。
- 每个 topic 顶部包含“用途、读取信号、关联 topic”。
- 经验按高频和高风险排序。
- 删除长期没有命中的经验。
