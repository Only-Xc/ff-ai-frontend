# 抽象与边界

用途：判断代码放在哪里、何时抽象、如何封装共享能力。

读取信号：wrapper、hook、shared component、页面模块拆分、公共工具、复用边界。

关联 topic：`third-party-components`、`layout-measurement`、`routing-page-structure`。

## 经验

### EXP-ABST-001 抽象先确认共同职责

- 触发：新增 wrapper、hook、共享组件，或把页面逻辑提升到公共模块。
- 经验：先确认多个调用点共享同一职责，再抽出最小稳定接口；页面差异由调用方传入。
- 检查：共享行为一致；输入输出稳定；调用方只传业务差异；页面专属配置留在页面侧。
- 关联：封装第三方组件读 `third-party-components`。

### EXP-ABST-007 避免无语义薄封装

- 触发：准备新增方法封装局部变量、`Set`/`Map` 基础操作、字段赋值或单个对象方法调用。
- 经验：新增方法前先确认它承载了稳定业务语义、副作用隔离、复用边界或错误处理；纯代理调用直接使用变量或底层 API。
- 检查：方法名是否表达了底层调用之外的业务意图；方法体是否只有一行代理；未来变更点是否真实集中；调用点直接写是否更清晰。
- 关联：无。

### EXP-ABST-008 局部分发策略就地声明

- 触发：同一个函数内根据事件名、命令、状态或类型分发到少量处理方法。
- 经验：优先在分发函数内声明局部映射对象，key 使用协议命令或业务枚举值，value 使用已有处理方法或箭头函数转发。
- 检查：映射是否只服务当前函数；key 是否和协议字段一致；value 是否保持 `this` 上下文；默认分支是否明确；新增事件是否只需补一行映射。
- 关联：无。

### EXP-ABST-002 重复配置建立单一来源

- 触发：父级容器和 children、tab、column、item 同时声明 `key`、`title`、`value`、权限或展示元数据。
- 经验：把业务配置集中在最接近声明的位置，父级负责归一化、默认值合并和内部结构生成。
- 检查：字段重复；默认值优先级；key 和顺序稳定；调用方显式声明得到保留。
- 关联：状态映射读 `state-data-flow`。

### EXP-ABST-003 第三方封装保留原生契约

- 触发：封装 AntD Table、Form、Modal、Select 或其他复杂第三方组件。
- 经验：封装层承载项目默认值和约束，同时透传原生 props、ref、className、style、事件和插槽。
- 检查：受控/非受控；ref；className/style 合并顺序；render props；loading、empty、disabled 状态。
- 关联：组件 API 读 `third-party-components`。

### EXP-ABST-004 页面专属配置就近维护

- 触发：columns、操作项、筛选 options、枚举展示或表单字段强绑定单页接口、权限、i18n 或交互动作。
- 经验：单页专属配置放在页面模块附近；跨页面稳定复用的纯映射、类型或工厂函数进入共享模块。
- 检查：是否依赖页面 action、权限、接口字段、局部状态；抽离后页面入口是否仍能读懂主流程。
- 关联：业务映射读 `state-data-flow`。

### EXP-ABST-005 页面模块按职责分层

- 触发：在页面目录新增辅助函数、常量、类型、格式化、筛选归一化、页面状态 hook 或请求适配逻辑。
- 经验：`components` 放可渲染 UI；非渲染逻辑放同层 `utils`、`constants`、`types`、`hooks` 或适配模块。
- 检查：文件是否返回 JSX；纯函数、常量、类型、状态 hook 是否进入对应目录。
- 关联：路由级页面结构读 `routing-page-structure`。

### EXP-ABST-006 通用逻辑优先查既有能力

- 触发：新增日期处理、集合转换、对象分组、键值索引、节流防抖、存储访问或浏览器 hook。
- 经验：先查项目已有工具和已安装依赖；已有能力匹配时复用；窄场景保留局部实现。
- 检查：项目导出；当前 package 依赖；tree-shaking；时区和 locale；空值边界；浏览器生命周期。
- 关联：验证和报告读 `verification-reporting`。
